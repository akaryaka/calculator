- проверить код на стандарты vue
- - Избегайте файлов > 300 строк. Выносите логику в composables/ или отдельные компоненты.
- - Предпочитайте ref() для единообразия. reactive() используйте только для объектов с фиксированной структурой.
- - Избегайте создания функций в шаблоне: @click="() => doSomething(id)" → используйте обработчики или computed/watch.
- - Vuex устарел для новых проектов. Используйте Pinia.
- - Храните в сторах только глобальное состояние (авторизация, настройки, кэш, корзина). Локальное состояние держите в компонентах/composables.
- - v-once для статичных компонентов, v-memo для списков с условным ререндером.
- - shallowRef / markRaw для больших неизменяемых данных (графики, таблицы, сторонние SDK).
- - keep-alive только для действительно часто переключаемых вкладок. Не злоупотребляйте.
- - Виртуализация для списков > 100 элементов (vue-virtual-scroller, @tanstack/vue-virtual).
- - Оптимизируйте computed: избегайте тяжёлых вычислений без кэширования, не мутируйте внешние состояния внутри них.
- - Используйте defineAsyncComponent для тяжелых UI-элементов (модалки, редакторы, графики).
- - Поддерживайте клавиатурную навигацию (Tab, Enter, Esc, фокус-стейты).
